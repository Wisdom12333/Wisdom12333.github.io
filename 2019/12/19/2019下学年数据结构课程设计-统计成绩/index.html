2019下学年数据结构课程设计-统计成绩

發表於 2019-12-19 | 分類於 随笔

选题及思路

这次课程设计主要是根据本学期学习的数据结构知识,做一个较为简单的程序.

主要选题

  • 线性表
  • 栈、队列
  • 串
  • 数组矩阵
  • 二叉树
  • 图
  • 查找排序

而在这七大类共31个小题内,我一眼就选中了”统计成绩”这个题目
具体题目如下图

思路

一开始,我想到的是用结构体存储,然后重载运算符,利用一次sort排序,即可完成对成绩的大小排序.
可是马上这个思路被我否决了,由于课程设计需要使用C语言,不能直接使用C++中的sort函数.其次,对于一个期末的总结性实践项目,最好是使用在课堂上学习到的内容.
然后我针对性的查找了教材的第10章”内排序”.很快我就找到了一个例题”学生信息多关键字排序”(P286实验题12).这个题目中,也是存储学生信息,最后根据性别或者班号来进行排序.参考它的写法,我大致有了我题目的思路:

使用结构体进行输入数据的存储—>创建单链表保存—>对链表进行快速排序—>输出

第一次成品

刚开始的程序十分简陋,由于需要的数据有学生人数,科目信息,学生具体信息.所以运行顺序为输入n(学生人数),m(科目数量).然后for循环输入各科目名称.最后依次输入学生的学号、姓名和各科成绩,同时计算总成绩,一起存入结构体StudType中.

之后使用尾插法创建单链表

最后就是排序了,这里花费了不少时间,主要是排序过程中两个结点内容的交换.
一开始一直困恼的是字符数组的交换,后来突然想起有strcpy函数,一开始一直想的是使用指针完成操作.然后就是直接交换还是使用函数,又因为对结构体指针不够熟悉,也是百度了很久,看了很多其他人的做法,终于才写出来.

之后的排序选择了快速排序算法,一是根据的教材,二是通过他人代码学习的单链表快速排序算法.

最后输出也是费了很多脑筋,想尽办法让打印出的名次表更好看.主要是控制输出的长度,使每一栏都可以占据同样的字符长度.代码就不放了,丢人.效果如下:

这也就是第一次的作品了,简单朴实.可是上课给老师看了一下,才发现存在很多问题.
首先是一开始的数据结构,成绩竟然是用的整形,我也是傻了.然后,老师说要做一个菜单功能,最后要设置数据边界,当输入数据错误时返回错误提示.
所以也就做了一些微小的改动.

第二次

照着之前的建议,首先增加了一个菜单功能.
思路是使用一个菜单函数,然后while循环函数,当输入特定的数时退出.
效果如下

看起来也还行,然后使用该函数读取的数组进行对应操作,就很简单了.
然后将输入进行修改,使用函数封装,就是一个有模有样的程序了.
可是运行的时候问题一下就出来了,首先是因为输入用了函数,就需要用指针或者全局变量,可是指针尤其是字符串的指针又一直搞不清楚,这里就琢磨了很久.
然后就是发现输出的时候会输出完崩溃程序,检查了很久,最后发现是链表的指针指向了NULL,而且存在将它赋值给别的整数的情况,导致了Bug的存在.
